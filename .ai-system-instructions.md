# About the application

The lgaicm (LazyGit AI Conventional Messages) script is a specialized Bash tool designed to generate and apply AI-powered conventional commit messages. It is specifically architected to integrate with Lazygit via custom commands, providing a seamless workflow for creating
  high-quality, multi-line commit messages based on staged changes.

  Core Functionality

  The script operates in two primary phases: Suggestion and Execution.

  1. The suggest Workflow
  This is the "Research and Propose" phase. When you run lgaicm suggest --type <type>, the script performs the following technical steps:

   * Context Gathering: It retrieves the current Git state using git diff --cached --stat (summary) and git diff --cached (full patch).
   * Safety Truncation: To prevent hitting API token limits or incurring excessive costs, it uses the truncate_bytes function. By default, it limits the diff stat to ~60KB and the full patch to ~200KB.
   * Prompt Engineering: It constructs a detailed prompt for the Gemini AI. The prompt enforces:
       * Conventional Commits: Strict adherence to the provided type (e.g., feat, fix).
       * Structure: Mandatory subject line and a detailed body explaining what and why.
       * Content: Instructions to include behavior changes, risks, and regression testing notes.
       * Format: It explicitly requests a raw JSON array: [{ "subject": "...", "body": "..." }].
   * API Interaction: It communicates with the Gemini Interaction API (v1beta/interactions) using curl. It handles the system_instruction separately from the user prompt for better model steerability.
   * Output Processing:
       * It uses jq to parse the JSON response.
       * It normalizes the subject lines (stripping trailing periods, ensuring the correct prefix).
       * It creates a temporary "Session Directory" (e.g., /tmp/lgaicm.XXXXXX) and writes each suggestion into an individual text file.
       * Finally, it prints each suggestion in a specific format: Subject <===> /path/to/message/file. This format is designed for easy parsing by Lazygit's menuFromCommand prompt.

  2. The commit Workflow
  This is the "Finalization" phase. When a user selects a suggestion in Lazygit, the script runs lgaicm commit --file <path>:

   * Execution: It runs git commit -F <path>, using the -F flag to read the multi-line message directly from the generated file.
   * Cleanup: Once the commit is successful, it identifies the session directory and deletes it to keep the system's /tmp folder clean.

  Detailed Technical Components

  API Configuration
  The script is highly configurable via environment variables:
   * LGAICM_MODEL: Defaults to gemini-3.1-flash-lite for speed and cost-efficiency.
   * LGAICM_CURL_TIMEOUT: Default 45s, ensuring the CLI doesn't hang indefinitely on poor connections.
   * LGAICM_MAX_SUBJECT_LENGTH: Ensures subjects stay within standard Git limits (default 500 characters, though prompts suggest wrapping).

  Normalization Logic (normalize_subject)
  This function is a safeguard. Even if the AI provides a slightly malformed subject, the script uses sed and tr to:
   1. Strip leading/trailing whitespace.
   2. Force the conventional type prefix (e.g., changing Add: ... to feat: ...).
   3. Remove trailing periods to comply with Git conventions.

  Lazygit Integration (lazygit-config.yml)
  The script is designed to be used in a customCommand like this:
   1. User triggers <c-a>: A menu asks for the commit type.
   2. AI Generation: Lazygit runs lgaicm suggest.
   3. Selection: The user sees the AI-generated subjects. Selecting one passes the hidden file path to the commit command.
   4. Completion: The commit is made and the UI refreshes.

  Error Handling and Debugging
   * die() and log_debug(): The script implements a robust logging system. All API requests, raw responses, and internal state changes are logged to /tmp/lgaicm_debug.log.
   * Validation: It validates the presence of git, jq, and curl, and ensures the user is inside a Git repository with staged changes before attempting any AI calls.
   * JSON Resilience: It includes logic to strip Markdown code blocks ( `json ... ` ) in case the AI ignores the "raw JSON only" instruction.
